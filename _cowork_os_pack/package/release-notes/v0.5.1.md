# CoWork OS v0.5.1 Release Notes

**Release Date:** 2026-03-18
**Previous Release:** v0.5.0 (2026-03-15)

This patch release covers 69 commits since `v0.5.0`, focused on connector consolidation, tool-catalog correctness, collaborative task UX, local AI, research channels, runtime polish, and follow-on fixes around remote access, WhatsApp, renderer behavior, and development startup.

---

## Highlights

- **Connector catalog consolidation** — shipped MCP connectors are now curated to the supported allowlist, with Google consolidated under `google-workspace`
- **Native-first GitHub and Notion routing** — CoWork now prefers direct API paths and falls back to MCP only when needed
- **Versioned tool catalog snapshots** — executor and MCP clients rebuild tool views immediately when catalog state changes
- **Collaborative UI polish** — inline agent headers, Lucide role icons, markdown normalization, and parent-task back-navigation for sub-tasks
- **Research channels** — Telegram and WhatsApp link-dump chats can now trigger structured findings reports automatically
- **HuggingFace Local AI** — new `hf-agents` + `llama.cpp` local provider with server lifecycle management
- **Runtime fixes** — improved reconnect/backoff behavior, WhatsApp session invalidation, clearer renderer output handling, and duplicate-write fixes

---

## Added

### Local AI & Research
- Add HuggingFace Local AI provider with `hf-agents` + `llama.cpp`
- Add research-channel routing for Telegram and WhatsApp chats
- Add the `cowork-multi-agent-research` skill resource bundle

### Tooling & Registry
- Add stable SHA-1 tool catalog versioning across native tools and MCP state
- Add immediate MCP tool snapshot rebuilds on status changes and `tools_changed`
- Add tests for tool catalog versioning, executor cache busting, and MCP client rebuild behavior
- Add registry metadata updates including routing examples and authoring complexity

### Collaboration & UX
- Add sub-task back-navigation from main task content
- Add markdown normalization helpers for collaborative output
- Add cleaner notification display with title badges, humanized statuses, and direct view actions
- Add auto-discovery of an available development port on startup

### Reliability & Coverage
- Add tests for completion candidate ordering, helper-cache behavior, X mentions backoff, startup handlers, and attachment content stripping

---

## Changed

### Connectors & Integrations
- Consolidate Google services under the shipped `google-workspace` connector
- Remove DocuSign, Outreach, and Slack from the shipped Tier-1 connector surface
- Prefer native CoWork paths for GitHub and Notion, with MCP as fallback
- Reframe GitHub, Notion, and Slack registry skills as workflow wrappers around native CoWork paths

### Collaborative Runtime & Renderer
- Refresh the collaborative panel and dispatched-agent rendering
- Replace raw emoji UI markers with Lucide role and action icons across collaborative surfaces
- Simplify the sidebar sessions header and tighten filter/layout behavior
- Improve renderer handling for initial prompts and strategy-context stripping

### Platform & Runtime
- Improve remote-client reconnect backoff and error reporting
- Downgrade expected localhost Ollama connection failures to info-level logs
- Refresh docs for the current post-`v0.5.0` platform state across README, features, getting-started, connectors, workflows, and showcase pages
- Bump key dependencies including `lucide-react`, `pdfkit`, `@anthropic-ai/sdk`, `@aws-sdk/*`, and `oxfmt`

---

## Fixed

- Auto-invalidate WhatsApp sessions when stored credentials become unreadable
- Allow overwriting a file at the same path without duplicate warning noise
- Prefer non-verification output as the final response when executor candidates compete
- Ensure the initial prompt is always shown in the renderer
- Apply exponential backoff and better failure classification for X mention polling
- Extract control-plane auto-connect logic cleanly and remove a legacy remote fallback path
