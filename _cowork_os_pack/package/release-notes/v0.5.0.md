# CoWork OS v0.5.0 Release Notes

**Release Date:** 2026-03-15
**Previous Release:** v0.4.13 (2026-03-05)

This is a major feature release with 463 commits across agent orchestration, autonomous self-improvement, workspace memory, managed devices, a new semantic timeline, company operations infrastructure, and more.

---

## Highlights

- **Sub-Agent Orchestrator** — Delegated execution lanes with confirmation gates, risk classification, and capability-based model selection
- **Autonomous Self-Improvement Loop** — Bounded improvement campaigns with multi-variant evaluation, promotion gates, cooldowns, and direct-apply review
- **Unified Memory Synthesizer** — All 6 memory subsystems merged into a single deduplicated, relevance-ranked context block
- **Workspace Kits** — Structured workspace memory with frontmatter linting, revision snapshots, and a lint CLI
- **Companies & Strategic Planner** — Zero-human company ops with automatic issue generation and company-scoped digital twin personas
- **Managed Devices & Remote Sessions** — Fleet connection manager, remote file picker, real-time remote task monitoring
- **Semantic Timeline** — Normalized event renderer with specialized cards for agent activity, approvals, and summaries
- **Mermaid Diagram Rendering** — Inline diagram steps routed through `create_diagram` and rendered safely in the task feed
- **Google Workspace MCP Connector** — Sheets, Docs, and Chat integration via a new connector marketplace UI

## Upgrade Notes

- **Plan mode rename**: "propose" mode has been renamed to "plan" mode throughout the UI and codebase. API consumers using `propose` mode string values should update to `plan`.
- **Azure OpenAI**: Tool lists are now hard-capped at 128 tools to match Azure OpenAI API limits.
- **Sandbox tools**: `execute_code` and related sandbox tools are hidden until E2B is configured in settings.
- **Self-improvement**: The improvement loop now requires owner enrollment before running. Enable via Settings → Automations → Self-Improve.
- **Docker**: The `COWORK_TZ` environment variable is now validated; invalid IANA values fall back to UTC with a warning.
